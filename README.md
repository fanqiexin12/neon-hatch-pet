# Neon Hatch | 赛博像素宠物舱

一个赛博像素电子宠物项目。v1 是网页宠物舱；v2 新增真正的桌面宠物窗口，宠物可以透明置顶显示在桌面上。

当前版本：`v2.11.0`。完整版本记录见 [CHANGELOG.md](./CHANGELOG.md)。

## 运行

### 桌面宠物版

首次运行先安装依赖：

```bash
npm install
```

启动桌面宠物：

```bash
npm run desktop
```

### 打包给离线电脑

离线版本的思路是：先在一台**联网的同系统电脑**上把 Electron 运行时和项目文件打成 zip，再把 zip 拷贝到离线电脑运行。

重要：

- 目标离线电脑不需要安装 Node.js、npm，也不需要联网。
- macOS 包建议在 macOS 上打；Windows 包建议在联网 Windows 上打。
- 打包机和离线目标机最好是同架构。例如 Apple Silicon Mac 对 Apple Silicon Mac，Windows x64 对 Windows x64。
- 每次改完功能后重新打包，zip 文件会带当前 `package.json` 里的版本号。

#### macOS

在联网 Mac 上先安装依赖，然后打包：

```bash
npm install
npm run package:mac
```

打包完成后会生成类似：

```text
release/neon-hatch-pet-v2.11.0-mac-arm64.zip
```

把这个 zip 拷贝到另一台同架构 Mac，解压后运行：

```text
Neon Hatch Pet.app
```

注意：

- 当前脚本打包的是本机 macOS 架构；Apple Silicon 常见输出是 `mac-arm64`，Intel Mac 常见输出是 `mac-x86_64`。
- 如果要给 Intel Mac 使用，推荐在 Intel Mac 上打包；如果要给 Apple Silicon Mac 使用，推荐在 Apple Silicon Mac 上打包。
- 这个包是本地自签名/未公证应用，macOS 可能弹出安全提示。可右键点击 app 选择“打开”；如果是从网络下载导致隔离标记，可在终端执行 `xattr -dr com.apple.quarantine "Neon Hatch Pet.app"`。

#### Windows

在一台联网 Windows 电脑上安装 Node.js/npm 和 Git，然后执行：

```powershell
git clone https://github.com/fanqiexin12/neon-hatch-pet.git
cd neon-hatch-pet
npm install
npm run package:win
```

打包完成后会生成类似：

```text
release/neon-hatch-pet-v2.11.0-win-x64.zip
```

把这个 zip 拷贝到离线 Windows，解压后运行：

```text
Neon Hatch Pet\Neon Hatch Pet.exe
```

注意：

- 目标离线 Windows 不需要安装 Node.js/npm，也不需要联网。
- 当前 Windows 打包脚本使用本机已安装的 Electron runtime，所以需要先在联网 Windows 上执行 `npm install`。
- 不要只拷贝 `Neon Hatch Pet.exe`，要拷贝整个解压后的文件夹，因为 Electron 还需要旁边的 `.dll` 和 `resources/` 文件。
- 没有代码签名时，Windows SmartScreen 可能提示“未知发布者”。自己使用可以点“更多信息”然后选择“仍要运行”；如果要长期分发给别人，建议做正式代码签名。
- `package:win` 会打包当前 Windows 的架构，常见是 `win-x64`；ARM Windows 会生成对应架构包。

桌面版特性：

- 透明、无边框、置顶的小窗口，只显示蛋或宠物本体
- 直接拖动宠物本体移动窗口；拖动结束后会随机给出搬家、巡航、升降等互动反馈
- 初始为蛋，点击蛋或右键菜单可以孵化；已孵化后可通过右键菜单“重孵为蛋”
- 支持喂食、玩耍、探险、摸摸、跳舞、睡觉、清洁、挠痒、喂零食、躲猫猫、合影和心情选项回应
- 点击像素宠物本体可以快速孵化或摸摸它
- 鼠标移到宠物身上会浮现分类底部互动栏；孵化后只保留照顾、心情和今日三类入口
- 底部互动栏支持喂食、玩耍、摸摸、探险、跳舞、睡觉、清洁、挠痒、零食、躲猫、合影、十种心情，以及今日 ToDo 的计划、反馈和总结
- 今日 ToDo 支持每行一件事的清单输入，也支持单项编辑、拖动排序、上移/下移、显示还剩几项和完成庆祝反馈
- 隔天未完成的 ToDo 会保留为“可延续项”，你可以选择一键延续到今天
- 右键菜单可设置 ToDo 早上询问时间、白天追问间隔和晚上总结时间
- 到早上设定时间时，宠物会询问今天要做什么；白天会追问进度；晚上会总结当天完成度
- 宠物颜色会随上午、中午、下午、晚上、夜里变化，并叠加春夏秋冬的季节色调
- 喂食、玩耍、摸摸、探险、跳舞、睡觉、清洁和心情回应都会根据时间段、当前状态、亲密阶段和季节随机生成不同反馈
- 每类互动都有多种随机像素动作，例如充电式喂食、轨道玩耍、探险地图、数据宝箱、贴贴、旋转跳舞、梦境睡眠、清洁闪光、挠痒笑点、零食彩蛋、躲猫猫和合影闪光
- 每天会随机安排 2-4 个陪伴时长提醒；宠物会告诉你已经陪伴主人多久，并使用符合时间段、陪伴阶段和季节的语气
- 更多互动集中在右键菜单中，不再显示背景面板、常驻按钮区或文字输入框
- 右键菜单支持关系设置：可以给宠物命名、设置主人称呼，并立即让宠物叫主人
- 宠物会在互动反馈里带上主人称呼，也会按设置的间隔随机呼唤主人
- 呼唤主人间隔支持关闭、2 分钟、5 分钟、10 分钟、30 分钟
- 右键菜单支持陪伴询问：宠物会按上午、中午、下午、晚上、夜里给出不同问候，问你现在在工作、看书、学习、娱乐、休息还是发呆
- 选择当前活动后，宠物会给出对应反馈，并在一段时间后提醒你休息、整理重点或收束娱乐时间
- 主动询问间隔支持关闭、10 分钟、20 分钟、30 分钟、1 小时
- 宠物会记住最近 7 天的活动选择；如果你反复在某个时间段选择学习、工作等，它之后会在反馈里提到这个习惯
- 新增桌宠日常任务：记录活动、学习复盘、工作后休息、摸摸、喂食、心情互动、鼠标手势、拖动宠物等都可能完成今日任务
- 完成日常任务会获得信用点、快乐/能量/亲密度奖励，并触发 `DONE` 像素动画和反馈气泡
- 右键菜单可以查看今日任务进度、刷新今日任务，并查看亲密成长等级
- 亲密度会推动关系等级，从 Lv.1 初识伙伴成长到 Lv.5 灵魂搭档，升级时会触发专属动作和文字反馈
- 不点击宠物时，也会识别鼠标横滑、上滑、下滑、折线和绕圈，并给出对应动作和文字反馈
- 不用输入文字，可在右键菜单里选择开心、伤心、焦虑、累了、生气、无聊、孤单、兴奋、平静、压力
- 心情选项会触发更丰富的文字反馈和动作：庆祝、呼吸、贴近陪伴、降温、贴贴、聚焦等
- 右键菜单支持报时设置：可立即报时、设置报时间隔、切换报时格式
- 报时间隔支持关闭、5 分钟、15 分钟、30 分钟、1 小时、2 小时
- 报时格式支持 `HH:mm`、`HH:mm:ss`、中文口语、日期 + 时间
- 报时时宠物会配合动作、粒子和时间反馈气泡
- 鼠标静止停在宠物身上 3 秒，会显示饱腹、快乐、能量三项详细状态
- 饱腹、快乐、能量会随时间缓慢变化；低于可设置阈值时会提醒你喂食、玩耍或睡觉
- 低状态提醒阈值可在右键菜单设置为 30%、40%、50%、60% 或 70%
- 孵化后会在空闲时自动随机做动作并弹出短文字：环顾、小跳、挥手、伸展、摇摆、探头、聆听、陪工、扫描、守护、好奇、小梦、碎碎念
- 不同活动会触发不同像素动画：食物块、激光点、爱心、音符、睡觉 Z、清洁泡泡和安慰贴近动作
- 本地保存桌面宠物状态

### 网页宠物舱

```bash
python3 -m http.server 5173
```

然后打开：

```text
http://127.0.0.1:5173
```

## 功能

- 桌面版：纯透明桌宠窗口，可拖动宠物本体、照顾/心情/今日悬停互动栏、右键菜单互动、状态保存、重孵回蛋、宠物/主人命名、随机呼唤主人、今日 ToDo 计划和总结、时间段陪伴询问、短期活动记忆、日常任务、亲密成长、鼠标滑动反馈、心情选项回应、可配置报时、悬停状态、低状态提醒和自动随机动作/文字
- 自定义孵化时间，倒计时完成后随机孵化宠物
- 随机宠物身份、性格和像素外观
- 宠物昵称、等级、经验和信用点会保存到本地
- 饱腹、快乐、能量、羁绊、清洁五项状态
- 喂食、玩耍、清洁、休息和训练会改变状态并记录日志
- 追光小游戏：限时点击画布里的发光核心，按命中奖励经验和信用点
- 外出探险：随机触发城市事件，记录发现地点并可能解锁装扮
- 补给商店可以购买强化能量饼、激光玩具和共情芯片
- 装扮系统：可购买或解锁星环、光冠、量子围巾并显示在像素宠物身上
- 每日任务会追踪喂食、玩耍、心情互动、训练、追光和探险，完成后可领取奖励
- 输入心情后，宠物会根据关键词给出反馈
- 暴露 `window.render_game_to_text()` 和 `window.advanceTime(ms)` 便于浏览器自动化验证
